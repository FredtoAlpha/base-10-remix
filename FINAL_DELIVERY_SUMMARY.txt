================================================================================
                            THE SOLUTION
================================================================================

PROBLEM: Module can't handle multiple "passes" of group creation on same level
EXAMPLE: Can't do "6°1+6°2 → 3 groups" THEN "6°3+6°4 → 4 groups" in one day

ROOT CAUSES:
  1. Action bar cluttered (8 buttons no hierarchy)
  2. Stats consuming 50% screen
  3. Step 2 is just checkboxes (no "regroupement" concept)
  4. No visual indication which pass user is on

SOLUTION: Groupings System (3 phases, 40 hours total)

Phase 1 (DONE ✅):     12 hours
├─ Data model
├─ 24 utility functions
├─ PropertiesService persistence
└─ Backward compatibility

Phase 2 (NEXT 🚧):     18.5 hours (3.5 days intensive)
├─ Step 2: Groupings Composer (compose multiple passes)
├─ Step 3: Per-grouping Config (configure independently)
├─ Step 5: Tabs + Reorganized Actions (clean UI)
├─ Stats Panel: Resize to 1/3 width
└─ Group Cards: Add context (code + range)

Phase 3 (AFTER ⏳):    9 hours (1 day)
